# IGESS v0.8 Human Edit Verification Design

## Context

IGESS has an implemented Agent Analyst workflow that can produce human-only
table recommendations. The roadmap also points toward a future tuning report,
but current code does not yet include `igess tune`. v0.8 therefore closes the
recommendation loop in a way that works now with `advice.json` and stays
compatible with future tuning artifacts.

## Goal

Add a deterministic acceptance loop for human-owned table edits:

1. Agent produces an advice or tuning-style proposal.
2. Designer edits Luban/Excel source tables by hand.
3. IGESS verifies the current exported values against the recommendation.
4. IGESS re-runs lint and simulation and writes an auditable report.

IGESS must never apply source table edits automatically.

## Scope

Included:

- `igess review-proposal`.
- `igess verify-edits`.
- Proposal extraction from `table_recommendations`, top-level
  `recommendations`/`changes`, and future `best_candidates[*].changes`.
- Exact, numeric-range, and descriptive-suggestion table checks.
- Optional source workbook export from `--datas` into the verification output
  directory.
- Fresh run artifacts under the verification output directory.
- Optional compare/gate artifacts when a baseline is supplied.

Deferred:

- Automatic Excel writes.
- Dashboard review queue.
- Multi-scenario acceptance suites.
- Persistent `.igess/decisions` history.

## Report Status

- `passed`: exact or range recommendations matched and simulation succeeded.
- `needs_review`: simulation succeeded, but at least one recommendation was
  descriptive and needs human judgment.
- `failed`: at least one checked value was missing/mismatched or a gate failed.

## Safety Rules

- Source `.xlsx` files are read-only.
- `--datas` export writes only under the verification output directory.
- Recommendations remain `apply_mode: human_only`.
- Every check reports table, row, field, expected value, actual value, and
  status.

## Acceptance Criteria

- `review-proposal` writes `proposal_review.json` and `proposal_review.md`.
- `verify-edits` writes `verification_report.json`, `verification_report.md`,
  and a fresh `run/` directory.
- `verify-edits --tables` verifies existing exported table JSON.
- `verify-edits --datas` exports source workbooks without modifying them.
- Existing IGESS workflows and tests remain compatible.
