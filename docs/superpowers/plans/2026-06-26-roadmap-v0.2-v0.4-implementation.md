# Roadmap v0.2-v0.4 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the v0.2-v0.4 roadmap so IGESS has static Web reports, a local dashboard, run comparison, parameter scanning, regression gates, and project ergonomics commands.

**Architecture:** Keep the simulation core stable and add feature modules around deterministic run artifacts. `OutputWriter` writes a `run_manifest.json`; reporting/dashboard/compare/scan/gate/doctor/explain consume filesystem artifacts through typed loaders instead of re-running hidden logic. CLI commands remain the primary integration surface, with dashboard using the same service modules.

**Tech Stack:** Python 3.12 stdlib, existing PyYAML/openpyxl/pytest dependencies, static HTML/CSS/JS with no frontend build system, stdlib `http.server` for the local dashboard.
